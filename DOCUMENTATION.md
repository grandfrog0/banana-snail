# 1. Введение и общая характеристика проекта
- Проект «Banana Snail» представляет собой двухмерный компьютерный игровой симулятор в жанре платформера-головоломки. Игра разработана на языке программирования Python с использованием специализированной графической библиотеки Arcade, основанной на Pyglet и OpenGL;
- Суть проекта: Игрок управляет персонажем-улиткой, цель которого — собрать все бананы на уровне, найти ключ и добраться до выходной двери. Проект ориентирован на демонстрацию возможностей кастомного программирования физики твердых тел, систем частиц, работы с UI-компонентами и мультимедиа-ресурсами.

# 2. Цели и задачи разработки
2.1. Цель разработки: создание полноценного игрового приложения с уникальной механикой перемещения, визуальной целостностью и системой сохранения игрового прогресса.

2.2. Основные задачи:
- Разработать алгоритм перемещения и прилипания персонажа к типам поверхностей, таким как пол, стены и потолок;
- Реализовать визуальные эффекты (анимация спрайтов, генерация динамических частиц) для улучшения игрового опыта;
- Интегрировать систему звуковых эффектов;
- Разработать архитектуру уровней на основе внешних конфигурационных файлов (JSON) и построить систему сохранения прогресса.

# 3. Технический стек и структура проекта
3.1. Проект имеет модульную архитектуру и реализован с использованием следующих технологий:
- Язык разработки: Python 3.8+
- Графический движок: Arcade (OpenGL).
- Формат хранения данных: JSON;
- Формат аудиоресурсов: MP3.
3.2. Структура каталогов:
- main.py — точка входа, логика главного меню, инициализация UI-компонентов и управление сохранениями;
- game.py — игровой цикл, кастомная физика игрока, система частиц и менеджер камеры;
- levels/ — каталог с картами уровней в формате матрицы;
- assets/ — структурированное хранилище графики (images/) и звуковых эффектов (sounds/).

# 4. Архитектура и ключевые особенности реализации
4.1. Кастомный физический движок и инверсия гравитации: в отличие от классических платформеров, где вектор гравитации направлен строго вниз, в «Banana Snail» реализована динамическая смена ориентации персонажа в пространстве:
- Механика прилипания: с помощью коллизий класс SnailPlayer ежекадрово определяет тип поверхности под собой;
- Трансформация спрайта: В зависимости от текущего состояния поверхности изменяется угол наклона спрайта, а также пересчитываются векторы прыжка и направления движения.

4.2. Динамическая система частиц
Разработан легковесный класс ParticleSystem. При переходе персонажа из состояния полета в состояние контакта с поверхностью, система считывает тип поверхности и мгновенно генерирует в точке контакта 5 частиц пыли, после чего автоматически удаляются из памяти.

4.3. Анимация
Наборы текстур у спрайта улитки разделены на левое и правое направления и содержат по 2 кадра анимации.

4.4. Звуковое сопровождение
Все ключевые триггеры игры подкреплены аудиоэффектами в формате .mp3.

4.5. Сериализация данных
Прогресс игрока не сбрасывается при закрытии приложения. При запуске игра считывает файл save.json. Прогресс сбора бананов кодируется битовой строкой (например, "101" — собран первый и третий банан).

# 5. Заключение
Игровое приложение «Banana Snail» успешно реализует все поставленные задачи. Разработанные модули физики, анимации и частиц являются гибкими и масштабируемыми, что позволяет в будущем легко расширять игру.
